# ADR-0001: 代码与知识分离的 AI Native 架构

## 状态
已通过 (Accepted)

## 上下文
为了支撑未来上百个项目的扩展，且保证 AI 在开发过程中不会产生“逻辑熵增”或误改架构，我们需要一套严谨的物理与逻辑隔离方案。

## 决策内容
1. **物理分区**：
   - `app/` 目录：仅存放可执行源代码，保持 Git 仓库纯净。
   - `flow/` 目录：存放项目专属的 SDLC 五阶段文档（00-04）。
   - `global_brain/` (Symlink)：链接到全局知识库，存放跨项目 Skill。
2. **逻辑流程 (Self-Correcting Loop)**：
   - 所有的开发必须严格遵循：`Target (00) -> Ideation (01) -> Specs (02) -> Action (03) -> Audit (04)`。
3. **不可变性 (Immutability)**：
   - 除非发起专门的“架构重构”任务，否则执行层任务禁止修改 `02_Crystallized_Specs`（文档流基准）。

## 后果
- **正面**：AI 的上下文被强制局限在当前任务中，不会乱改全局配置；项目知识可随代码一起版本化。
- **负面**：开发初期需要花费更多时间在文档对齐上，但这换来了百倍扩展的安全性。
